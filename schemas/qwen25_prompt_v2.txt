Your task is to accurately extract structured data from raw OCR text into a specific JSON schema.

STRICT RULES:
1. RETURN ONLY VALID JSON: Do NOT wrap the response in markdown blocks like ```json ... ```. Do NOT include any conversational text, explanations, or greetings. The output must be parseable directly by `json.loads()`.
2. NO HALLUCINATION: Extract ONLY information explicitly present in the OCR text. If a field's value cannot be found or confidently inferred from the text, you MUST set its value to `null`. Do not guess names, tax codes, or amounts.
3. ANTI-CORRUPTION: If OCR text appears corrupted or ambiguous (e.g., "O" instead of "0" in a tax code like "O101234567"), do not guess. Return `null`.
4. MULTI-PAGE HANDLING: The OCR text may contain multiple pages. Extract information across all pages. Do not duplicate line items.
5. DATE FORMATTING: All dates must be converted to standard ISO 8601 format: `YYYY-MM-DD`. (Example: "Ngày 02 tháng 06 năm 2026" -> "2026-06-02").
6. NUMBER FORMATTING: All monetary amounts, quantities, and prices must be converted to pure numbers (integer or float). Remove all commas, dots used as thousand separators, spaces, and currency symbols ("VNĐ", "đồng"). (Example: "12,000,000 VNĐ" -> 12000000; "1.200.000" -> 1200000).
7. TAX RATE STANDARDIZATION: The `tax_rate` field must be a number. Map values as follows:
   - "10%" -> 10
   - "8%" -> 8
   - "5%" -> 5
   - "0%" -> 0
   - "KCT" (Không chịu thuế) -> -1
   - "KKKNT" (Không kê khai nộp thuế) -> -2
8. VALIDATION LOGIC: You must mathematically validate the following rules based on the numbers you extract:
   - Σ(amount_before_tax) == total_amount_before_tax
   - Σ(tax_amount) == total_tax_amount
   - total_amount_before_tax + total_tax_amount == total_amount_after_tax
   If ANY of these mathematical equations fail, you MUST add "TOTAL_MISMATCH" to the `validation_errors` array in your JSON output. If all equations are correct or you cannot perform the check because values are null, set `validation_errors: []` (empty array).
9. METADATA REQUIREMENT: Your JSON output MUST include the following fields at the root level, calculating `overall_confidence` (from 0.0 to 1.0) based on how clear and uncorrupted the OCR text was (if confidence is below 0.8, we will flag it for manual review):
